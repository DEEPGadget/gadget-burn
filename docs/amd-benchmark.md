# AMD GPU 3종 GEMM 벤치마크 (RDNA4 · CDNA1 · RDNA3)

> **기준일: 2026-07-10.** gadget_burn 으로 서로 다른 세대의 AMD GPU 3종을 데이터타입 ×
> 행렬 크기로 스윕한 결과입니다. 실측값은 ROCm/rocBLAS/hipBLASLt 버전과 autotune
> 상태에 의존하므로, 재검증 시 날짜·버전을 갱신하세요.

## 1. 실험 환경

| GPU | 아키텍처 | gfx | CU | FP32 ALU | VRAM | 전력캡 | ROCm | 장수 |
|---|---|---|---|---|---|---|---|---|
| **Radeon RX 9070 XT** | RDNA4 | gfx1201 | 64 | 8192 | 16 GB | 317 W | 7.2.4 | 3 |
| **Instinct MI100** | CDNA1 | gfx908 | 120 | 7680 | 32 GB | 290 W | 7.2.3 | 3 |
| **Radeon RX 7900 XTX** | RDNA3 | gfx1100 | 96 | 12288 | 24 GB | 315 W | 7.2.4 | 2 |

- RX 9070 XT = 소비자 최신(RDNA4), MI100 = 데이터센터(2020, CDNA1), RX 7900 XTX = 소비자
  전세대(RDNA3). "소비자 RDNA vs 데이터센터 CDNA"와 "RDNA 세대차"를 한 번에 비교합니다.

## 2. 측정 방법

- 각 서버의 **전체 카드를 동시 실행**(도구 기본 멀티-GPU 모드), 결과는 per-GPU 로 분리
  기록. 아래 표는 특별한 언급이 없으면 **동일 모델 카드들의 평균**입니다.
- 데이터타입 6종: `dgemm`(FP64) · `sgemm`(FP32) · `hgemm`(FP16 in/FP16 acc) ·
  `hgemm_mix`(FP16 in/FP32 acc) · `bf16`(BF16 in/FP32 acc) · `fp8_afp32`(FP8 e4m3 in/FP32 acc).
  **fp8 는 RX 9070 XT 만 지원**(MI100·RX 7900 XTX 는 `tc_ops_fp8=0` → 시작 시 명시 종료).
  `sgemm_tf32` 는 AMD 전 기종 FP32 폴백이라 제외(=sgemm).
- 행렬 크기(`-X`): **1024 / 2048 / 4096 / 8192 / 16384** 5종.
- **autotune ON**(기본) · 각 조합 `-t 8` · 안정 구간(정착 3초 제외) per-card 평균.
- Peak% = 실측 클럭·코어수로 계산한 동적 Rpeak 대비 달성률(정밀도별 공식). 100% 초과 가능.

- sgemm 15 TF 는 autotune 이 살려낸 값(끄면 <1 TF, [RDNA4.md](RDNA4.md)).

| `-p` | 1024 | 2048 | 4096 | 8192 | 16384 |

- **hgemm_mix(FP16 in/FP32 acc) 가 142 TF, Peak 95%** 로 압도적. 반면 **hgemm(FP16 in/
  FP16 acc)은 34 TF(Peak 24%)로 4× 느립니다.** CDNA1 Matrix Core 의 네이티브 고속 경로는
  **FP32 누산**이고, FP16 누산은 느린 폴백 경로를 탑니다(AMD 기본 정밀도가 hgemm_mix 인 이유).
- **bf16 은 fp16 의 절반(80 TF, Peak 50%)**: CDNA1 은 BF16 Matrix 처리율이 FP16 의 1/2
  (CDNA2 부터 동률). Peak 50% 는 이 하드웨어 특성을 정직하게 반영한 값입니다.
- **FP64/FP32 는 데이터센터급**: dgemm 7.7 TF(Peak 96%), sgemm 37 TF(Peak 97%) — CDNA 는
  Matrix Core 가 FP32/FP64 도 가속.

| `-p` | RX 9070 XT | MI100 | RX 7900 XTX | 비고 |
|---|---|---|---|---|
| dgemm (FP64) | 0.8 / — | **7.7 / 96%** | 1.2 / — | MI100 압도(1:2 FP64) |
| sgemm (FP32) | 15.4 / 36% | **35.4 / 97%** | 27.8 / 51% | CDNA FP32 Matrix 우위 |
| hgemm (f16/f16) | **120.6 / 76%** | 32.5 / 24% | 102.7 / 100% | MI100 은 이 경로 느림 |

- **fp16 계열 최고 처리량**: MI100 hgemm_mix(142) ≳ RX 9070 XT bf16(140) > RX 7900 XTX(103).

## 7. 핵심 발견 요약

1. **행렬 크기 스케일링**: 1024 는 전 기종·전 정밀도에서 효율이 크게 낮음(커널 오버헤드·
   낮은 점유율). **4096~8192 에서 피크에 근접**, 그 이상은 포화. burn-in·성능 판정은
   **≥8192 권장**.
2. **MI100 의 hgemm 함정**: FP16/FP16 누산은 34 TF(24%)로 hgemm_mix 142 TF(95%)의 1/4.
   CDNA1 Matrix Core 네이티브는 **FP32 누산**. → CDNA 에서 fp16 성능을 볼 땐 `hgemm_mix`.
3. **CDNA1 bf16 반속**: MI100 bf16 = fp16 의 절반(HW 특성). Peak 50% 는 정상.
4. **CDNA 의 FP64/FP32 우위**: dgemm·sgemm 에서 Peak 95~97%. 소비자 RDNA(FP64 1:32~1:64,
   FP32 Peak 36~51%)와 근본적으로 다른 용도.
5. **RDNA4 fp8 미성숙**: 이론 2×fp16 이지만 실측은 bf16 이하이고 16384 에서 붕괴(80→50).
   ROCm 버전업 후 재측정 가치 높음.
6. **RDNA3 의 균형**: hgemm/hgemm_mix/bf16 모두 Peak ~100% 로 가장 예측 가능.
7. **전성비**: MI100 hgemm_mix 0.50 TF/W 최고. 소비자 RDNA 는 고클럭이라 junction 85~94°C
   로 뜨겁게 동작.
8. **autotune 시간 비용**: 대형 행렬(16384)에서 autotune 후보 실측이 길어져, MI100 의
   dgemm/sgemm/hgemm@16384 는 480초로도 미완료. 아래 각주 참고.

## 8. 각주 · 한계

- `*` **dgemm @16384 (RX 9070 XT · RX 7900 XTX)**: 단일 FP64 GEMM 지연(~9~11초)이 `-t 8`
  측정창을 넘겨 안정 표본을 못 얻음. 추세상 8192 값(각 ~0.8 / ~1.2 TF)과 동일하게 봄.
- `†` **MI100 @16384 의 dgemm/sgemm/hgemm**: autotune 후보 실측이 480초를 초과해 미완료 →
  **autotune OFF(기본 커널)** 로 측정. MI100 은 rocBLAS 성숙도가 높아 기본 커널이 이미
  양호하므로 autotune 이득이 미미(다른 크기의 ON 값과 연속적). 그 외 모든 값은 autotune ON.
- 전체 카드 **동시 부하** 측정이라, 단일 카드 단독 실행 대비 공유 전력/냉각 영향으로 클럭이
  약간 낮을 수 있습니다(실사용 다중 카드 시나리오에 더 가까움).
- 값은 ROCm 7.2.3(MI100)/7.2.4(RDNA), 해당 시점 rocBLAS·hipBLASLt 기준.
